25 ## Creating a custom command
26 Creating a custom command is a simple two step procedure.i
27 
28 1. We need to get the dialogue object that's in use, then regiser a new function for it.
29 ```
30  // get the Dialogue object
31  // varstorage_implementation is the object that handles variable storage - it's not important in this example
32  var dialogue = new Dialogue(varstorage_implementation);
33 ```
34 2. Next, register your new function. For example, let's make a function that takes 1 parameter, which is a number, and returns `true` if it's even:
35 ```
36  // RegisterFunction(name, parameterCount, implementation)
37  dialogue.library.RegisterFunction ("is_even", 1, delegate(Value[] parameters) {
38  return (int)parameters[0].AsNumber % 2 == 0;
39  });
40 ```
41 When the function is called, the delegate you provide will be run.
42 
43 Some notes:
44 
45 * You don't have to return a value from your function.
46 * The parameters passed to your function are of type Yarn.Value. You can get their value as numbers, bools or strings by using the `AsNumber`, `AsString` and `AsBool` properties.
47 * You can only return values of the following types:
48  * String
49  * Float
50  * Double
51  * Integer
52  * Yarn.Value
53 
54 * Yarn Spinner will make sure that the correct number of parameters is passed to your method. If you specify `-1` as your parameter count, the function may have any number of parameters.
