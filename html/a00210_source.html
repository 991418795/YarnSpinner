1 # Yarn Dialogue - General Usage Guide
2 
3 This document talks about how to use Yarn if you're using it to write content. It doesn't talk about how to integrate Yarn Spinner into your project; for that, see ["Using Yarn Spinner in your Unity game"](../YarnSpinner-Unity).
4 
5 ## Lines
6 
7 Yarn lets you define multiple chunks of dialogue that are linked together. Each chunk is called a *node*.
8 
9 Nodes are filled with different kinds of text. The simplest is lines of dialogue, which look like this:
10 ```
11 Alice: Hey, I'm a character speaking in a dialogue!
12 Bob: Wild!
13 ```
14 Each line is delivered to your game one at a time.
15 
16 ## Options
17 
18 To link a node to another node, you provide *options*. Options look like this:
19 ```
20 [[Go to the woods|GoToWoods]]
21 [[Go back to the city|GoToCity]]
22 ```
23 Options have two parts: the label, which is shown to the user, and the name of the node that we should link to when the user selects the option. The label and the node name are separated by a vertical bar (`|`).
24 
25 You don't have to provide the label. Instead, you can just provide the node name, like so:
26 ```
27 [[GoToCity]]
28 ```
29 If an option has no label, Yarn will automatically jump to that node when that line is run, and it won't show options to the player.
30 
31 When all of the lines in a node have been displayed, the dialogue system gives your game the list of available options. You then let the user choose an option, and then continue loading lines of dialogue, one at a time.
32 
33 If there are no options when we reach the end of a node, then we've reached the end of a conversation, and Yarn Spinner will let your game know.
34 
35 ## Shortcut Options
36 
37 Sometimes, you'll want to add little branches to your conversation, but you don't want to create separate nodes for them. Instead, you can use *shortcut options*:
38 ```
39 Mae: What did you say to her?
40 -> Nothing.
41  Mae: Oh, man. Maybe you should have.
42 -> That she was an idiot.
43  Mae: Hah! I bet that pissed her off.
44 Mae: Anyway, I'd better get going.
45 ```
46 When this is run, Yarn Spinner will behave just as if you'd broken all of this up into multiple nodes.
47 
48 You can also attach conditions to shortcut options, which will make them only appear if a certain condition passes:
49 ```
50 Bob: What would you like?
51 -> A burger. <<if $money >= 5>>
52  Bob: Nice. Enjoy!
53 -> A soda. <<if $money >= 2>>
54  Bob: Yum!
55 -> Nothing.
56 Bob: Thanks for coming!
57 ```
58 Note that in the last example, there wasn't any attached text. If the player selected the last option ("Nothing."), then the next line to appear would be "Bob: Thanks for coming!"
59 
60 You can also nest these shortcut options, if you like. Be careful, though - too much nested options can make your text difficult to read.
61 
62 ## Commands
63 
64 In addition to showing lines of dialogue, your game will probably want to run actions - things like "move the camera to position X", or "make character start smiling". You can use commands for this.
65 
66 Commands look like this:
67 ```
68 <<move alice to under_bridge>>
69 ```
70 Any text inside the double-chevrons will be sent directly to your game as a string. It's up to you to decide what to do with that string.
71 
72 Yarn has a special command, called `stop`. If you include `<<stop>>` in your node, Yarn will stop the entire conversation when it reaches it.
73 
74 ## Variables and If statements
75 
76 You can store numbers in variables. To do this, use the `set` command:
77 ```
78 <<set $door_unlocked to 1>>
79 ```
80 You can check the value of a variable using the `if` command:
81 ```
82 <<if $door_unlocked is 1>>
83  The door is unlocked! (This will only appear if $door_unlocked is equal to 1.)
84 <<endif>>
85 ```
86 ## Expressions
87 
88 Maths is easy in Yarn.
89 ```
90 << set $number_of_stars_collected = 1 >>
91 << set $number_of_stars_collected = $number_of_stars_collected + 1 >>
92 ```
93 > ***Note:*** `to` and `=` are synonyms. It is up to you which you use. However it is strongly recommended to maintain consistency through your Dialogue for readability purposes. At the end of this document, you will find a [list of operator synonyms](#operator-synonyms).
94 
95 [Order of operations](https://en.wikipedia.org/wiki/Order_of_operations) is as expected, but usage of brackets is encouraged for readability purposes.
96 ```
97 << set $globular_clusters_collected = 2 >>
98 << set $number_of_stars_collected = $number_of_stars_collected + ( $globular_clusters_collected * 1000 ) >>
99 ```
100 You can also do maths inside an `if` command. For example:
101 ```
102 <<if $number_of_stars_collected > 5>>
103  You have more than 5 stars!
104 <<endif>>
105 ```
106 You can also get fancier, and do stuff like this:
107 ```
108 <<if $hostages_saved == $number_of_hostages and $time_remaining > 0>>
109  You win the game!
110 <<elseif $hostages_saved < $number_of_hostages and $time_remaining > 0>>
111  You need to rescue more hostages!
112 <<elseif $bomb_has_exploded == 0>>
113  You failed to rescue the hostages before time ran out!
114 <<endif>>
115 ```
116 ## Types
117 
118 There are four different types of value in Yarn: strings, numbers, [booleans](http://eesemi.com/boolean.htm), and `null`.
119 
120 * Strings contain text.
121 * Numbers are floating-point numbers.
122 * Booleans are either true or false.
123 * `null` means no value.
124 
125 Yarn will automatically convert between types for you. For example:
126 ```
127 <<if "hi" == "hi">>
128  The two strings are the same!
129 <<endif>>
130 
131 <<if 1+1+"hi" == "2hi">>
132  Strings get joined together with other values!
133 <<endif>>
134 ```
135 
136 ## Functions
137 
138 You can call functions in your `if` commands. For example, Yarn Spinner provides a function called `visited`, which you can use to find out if a node has been entered before or not. It takes a single parameter (which is a string), and returns true or false depending on whether or not the node you specified has been entered.
139 ```
140 <<if visited("GoToCity")>>
141  We have gone to the city before!
142 <<endif>>
143 ```
144 You can't define your own functions inside Yarn itself, but they can be added at run-time. See ["Extending Yarn Spinner"](../YarnSpinner-Programming/Extending.md) for more info.
